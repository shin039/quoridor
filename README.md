# Quoridor

---

## ゲームのルール

### 基本ルール

- 9×9 のボードを使用（本実装は2人対戦）
- **Player 1**: 上辺（行0）からスタート → 下辺（行8）がゴール
- **Player 2**: 下辺（行8）からスタート → 上辺（行0）がゴール
- 各ターン、以下のどちらか1つを実行する
  1. **ポーンを移動する**（上下左右1マス）
  2. **壁を1枚設置する**（水平または垂直、2マス分）

### 移動のルール

- 上下左右の隣接マスに1マス移動できる
- 相手のポーンに隣接している場合、その向こうへ**飛び越し**ができる
  - 飛び越した先が壁や盤端の場合は、**横に飛び出す**（斜め移動）ことができる

### 壁のルール

- 各プレイヤーは最初に**10枚**の壁を持つ
- 壁は2マス分の幅を持ち、セルとセルの境界に置く
- 壁同士の**重なり・交差**は禁止
- 壁を置いても、**どちらのプレイヤーもゴールへの経路が残っていなければならない**（完全封鎖禁止）

### 勝利条件

自分のポーンを相手側のゴールラインに最初に到達させたプレイヤーの勝利。

## 動作環境

- Python 3.10 以上

## 01_console — コンソール版

### 起動

```bash
python 01_console/quoridor.py
```

### 画面イメージ

```
      0   1   2   3   4   5   6   7   8
    +---+---+---+---+---+---+---+---+---+
  0 |   |   |   |   | 1 |   |   |   |   |
    +---+---+---+---+---+---+---+---+---+
  1 |   |   |   |   |   |   |   |   |   |
    +---+---+===+===+---+---+---+---+---+
  2 |   |   |   # 2 |   |   |   |   |   |
    ...
  P1 [1]: 位置(4,0) 残り壁:9枚  ゴール→行8
  P2 [2]: 位置(4,2) 残り壁:9枚  ゴール→行0
```

- `===` : 水平壁
- `#`   : 垂直壁

### コマンド

| コマンド | 説明 | 例 |
|---|---|---|
| `move <col> <row>` | ポーンを移動 | `move 4 1` |
| `wall h <col> <row>` | 水平壁を設置 | `wall h 3 4` |
| `wall v <col> <row>` | 垂直壁を設置 | `wall v 4 3` |
| `help` | ヘルプ表示 | |
| `quit` | 終了 | |

### 壁の座標指定

```
水平壁 h (col, row) : 行 row と row+1 の間に横向きに設置
                      列 col と col+1 の2マスをまたぐ
垂直壁 v (col, row) : 列 col と col+1 の間に縦向きに設置
                      行 row と row+1 の2マスをまたぐ
col・row はいずれも 0〜7 の範囲
```

## 02_gui — GUI版（ローカル2人対戦）

### 起動

```bash
python 02_gui/quoridor_gui.py
```

### 操作方法

右側のパネルでアクションを選択してからボードをクリックします。

| モード | 操作 |
|---|---|
| **ポーン移動** | 緑にハイライトされた移動可能マスをクリック |
| **水平壁を置く** | 行と行の隙間にカーソルを合わせ（黄色プレビュー）クリック |
| **垂直壁を置く** | 列と列の隙間にカーソルを合わせ（黄色プレビュー）クリック |

### 画面の見方

| 表示 | 意味 |
|---|---|
| 青色の行（行0） | Player 2 のゴールライン |
| 赤色の行（行8） | Player 1 のゴールライン |
| 緑のハイライト | 移動可能なマス |
| 黄色のプレビュー | 設置予定の壁 |
| オレンジの太線 | 設置済みの壁 |
| 白い点線リング | 現在のターンのポーン |

## 03_network — ネットワーク対戦版

サーバー1台とクライアント2台で動作します。  
ゲームの状態管理・バリデーションはすべてサーバー側で行います。

### アーキテクチャ

```
クライアントA (Player 1)
        │  JSON / TCP
        ▼
    サーバー  ←── ゲーム状態を一元管理
        │         アクション検証・ブロードキャスト
        ▼  JSON / TCP
クライアントB (Player 2)
```

### 通信プロトコル

改行区切りのJSON（`\n` デリミタ）を使用します。

**サーバー → クライアント**

```jsonc
// プレイヤー番号の割り当て
{ "type": "assign", "player": 1 }

// ゲーム状態の配信（全アクション後にブロードキャスト）
{
  "type": "state",
  "pos":     { "1": [4, 0], "2": [4, 8] },
  "walls":   { "1": 10, "2": 10 },
  "h_walls": [[3, 4]],
  "v_walls": [],
  "turn":    2,
  "winner":  null
}

// エラー通知
{ "type": "error", "msg": "経路が塞がれます" }

// 情報メッセージ
{ "type": "info", "msg": "Player 2 の接続を待っています..." }
```

**クライアント → サーバー**

```jsonc
// ポーン移動
{ "type": "move", "col": 4, "row": 1 }

// 壁の設置
{ "type": "wall", "wtype": "h", "col": 3, "row": 4 }
{ "type": "wall", "wtype": "v", "col": 4, "row": 3 }
```

### 起動方法

#### 同じPC内で試す場合

```bash
# ターミナル1：サーバー起動
python 03_network/quoridor_server.py

# ターミナル2・3：クライアントを2つ起動
python 03_network/quoridor_client.py
python 03_network/quoridor_client.py
# → 接続ダイアログで localhost / 5555 を入力
```

#### 別々のPCで対戦する場合（同一LAN内）

```bash
# サーバー側PC
python 03_network/quoridor_server.py

# クライアント側PC（サーバーのローカルIPを指定）
python 03_network/quoridor_client.py
# → 接続ダイアログで 192.168.x.x / 5555 を入力
```

#### インターネット越しに対戦する場合

**ngrok を使う方法（最も手軽）**

```bash
# サーバー側：サーバーを起動
python 03_network/quoridor_server.py

# サーバー側：ngrok でトンネルを開く
ngrok tcp 5555
# → 表示されたアドレス（例: 0.tcp.ngrok.io:12345）を相手に伝える

# クライアント側：表示されたホスト・ポートで接続
```

**Tailscale を使う方法（定期対戦向け）**

```bash
# 両者が Tailscale をインストール・サインイン
# サーバー側の Tailscale IP（例: 100.x.x.x）を相手に伝える
# クライアント側：その IP / 5555 で接続
```

### ポート番号の変更

デフォルトは `5555` ですが、引数で変更できます。

```bash
python 03_network/quoridor_server.py 8888
# クライアント側で接続時に 8888 を指定
```

### クライアントの操作方法

GUI版と同じ操作方法です。  
**自分のターン以外はボードの操作が無効になります。**
